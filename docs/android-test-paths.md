# Android Test Paths

本项目 Android 自动化测试按业务路径覆盖，而不是只按底部 Tab 截图覆盖。

## Route Map

- `/home`: 首页仪表盘、今日待办、最近账单、快捷入口。
- `/items`: 事项列表、事项状态、完成入口。
- `/items/new`: 新建事项、模板、事项类型、分类、日期、金额、重复、备注。
- `/items/:id`: 事项详情、编辑、完成、删除入口。
- `/items/:id/edit`: 编辑事项并保存。
- `/bills`: 账单列表、月度收支、月份切换。
- `/bills/new`: 新建账单、收入/支出、分类、金额、日期、备注。
- `/bills/:id/edit`: 编辑账单、删除账单。
- `/statistics`: 收支统计、事项统计。
- `/settings`: 通知权限入口、导出、导入。

## Automated Coverage

`integration_test/app_smoke_test.dart` 是主业务回归测试，覆盖：

- 首页启动与首页文案。
- 事项 Tab 打开、新建普通事项、详情、编辑保存。
- 使用模板创建不同业务类别事项：到期提醒、工资收入、普通待办。
- 逐个覆盖所有快捷模板：交水电费、房租、宽带续费、会员订阅、保险续费、证件到期、药品过期、食品过期、滤芯更换、工资收入、普通待办。
- 选择事项分类、打开日期选择器并确认日期。
- 创建带金额的支出事项，并通过“完成并记账”生成账单。
- 从事项列表卡片直接“完成并记账”，覆盖根页面不能被错误 pop 的回归。
- 创建重复事项，验证每周、自定义每 N 天字段可用。
- 重复事项“完成并生成下一轮”。
- 带金额重复事项“完成并生成下一轮”并同步记账。
- 事项延期入口。
- 事项删除。
- 账单 Tab 打开、新建支出账单、编辑账单。
- 新建收入账单，验证收入展示。
- 账单分类选择、日期选择器、月份左右切换。
- 删除一条独立账单，验证删除链路。
- 统计 Tab 打开，验证收支统计和事项统计入口。
- 设置 Tab 打开，验证导入取消、导入失败、空数据导入成功、导出提示。

`.maestro/full_pages.yaml` 是设备级截图路径，覆盖每个主要页面并输出截图。
截图统一归档到 `screenshot/`，该目录被 Git 忽略。

## Manual Risk Areas

以下路径仍建议定期人工或专项自动化验证：

- Android 系统通知权限弹窗，因为它依赖系统版本和模拟器权限状态。
- 日期选择器跨月份、跨年份边界。
- 大量事项和账单下的长列表滚动性能。
